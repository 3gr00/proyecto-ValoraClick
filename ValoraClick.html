Contexto del Proyecto

Misión: Encuestas Académicas Inteligentes

🏫 El Escenario Real

En la mayoría de centros educativos, la evaluación de la satisfacción del alumnado se hace de forma puntual mediante papel o formularios aislados (ej. Google Forms). No hay estructura común, ni histórico, ni control de accesos.

Durante estas 12 semanas, desarrollaréis ValoraClick, una plataforma multicentro moderna para generar encuestas, acceder vía códigos QR y consolidar estadísticas en tiempo real.

❌ El Pasado: Datos Dispersos

📄
  • ⚠️ Formularios aislados sin histórico consolidado.
  • ⚠️ Imposibilidad de escalar a múltiples centros o cursos.
  • ⚠️ Resultados poco explotables para análisis longitudinal.
Impacto crítico: Al no existir una base de datos estructurada, comparar la satisfacción entre el trimestre 1 y el trimestre 3, o entre distintos cursos, requiere horas de trabajo manual cruzando hojas de cálculo.

✅ El Futuro: ValoraClick

🚀
  • ✔️ Plataforma web multicentro con backend robusto en Spring.
  • ✔️ Base de datos NoSQL (MongoDB) para máxima escalabilidad.
  • ✔️ Acceso instantáneo de alumnos mediante QR y exportación CSV.
Tu Misión: Diseñar un modelo de datos escalable, reglas de seguridad robustas y una App intuitiva preparada para futuras oleadas de desarrollo.

Alcance Funcional (MVP)

Requisitos mínimos viables que garantizan el funcionamiento de la plataforma en un entorno educativo multicentro.

Estructura Multicentro (MongoDB)

La elección de MongoDB (Documentos NoSQL) es perfecta para soportar esta jerarquía altamente anidada y dinámica:

  • 📁 Centros Educativos
  • └─ 📁 Niveles (FP, ESO)
  •   └─ 📁 Cursos (DAM, DAW)
  •     └─ 📁 Grupos (1º, 2º)
  •       └─ 📁 Asignaturas
  •         └─ 📄 Encuestas (Trimestrales)
  •           └─ 📄 Respuestas Anónimas
🍃

Flexibilidad de Documentos

Con MongoDB y Spring Data, podréis gestionar encuestas con preguntas dinámicas sin tener que alterar el esquema de tablas tradicional constantemente.

Administrador

Gestión global. Configura centros, cursos, ve todos los resultados consolidados y exporta CSV.

Profesor

Acceso limitado por Spring Security. Solo consulta resultados de sus asignaturas y genera los códigos QR.

Alumno

Acceso público/anónimo. Escanea el QR, responde la encuesta y envía los datos sin registrarse.

Haz clic en los roles para visualizar su matriz de permisos teórica.

Cronograma Global: 12 Semanas

El proyecto se divide en 4 fases orientadas a una entrega de valor metodológica y evitar bloqueos entre equipos.

F1

Análisis y Diseño Global (Semanas 1-2)

Estructura MongoDB, Wireframes de React, y contratos JSON (Swagger) de la API de Spring definidos.

F2

Construcción del Core (Semanas 3-6)

BD poblada, endpoints Spring respondiendo, Login React activo, QR generado y encuestas visibles.

F3

Integración y Mejoras (Semanas 7-9)

Seguridad real (JWT roles), gráficos operativos, exportación CSV desde backend y UX refinada.

F4

Estabilización y Cierre (Semanas 10-12)

Documentación técnica, pruebas E2E finales, manuales de usuario y presentación formal.

Distribución de Tiempo

Planificación Detallada Paralela (Squads)

El secreto de trabajar 20 personas es que en la Fase 1 defináis "contratos" (ej. cómo será el JSON de respuesta). Así, React puede usar datos falsos (Mocks) mientras Spring Boot y Mongo construyen la realidad por debajo.

Semanas 1-2: Acordar contratos y diseños

🗄️ BBDD (Mongo)
  • Diseño del esquema de colecciones y documentos.
  • Definición de jerarquías (Centros > Cursos).
  • Crear scripts JSON de datos iniciales.
⚙️ Backend (Spring)
  • Diseño de API REST (Swagger).
  • Setup del proyecto Spring Boot y Maven.
  • Entregar contratos JSON a Frontend.
🖥️ Frontend (React)
  • Wireframes UI/UX de la intranet.
  • Crear proyecto Vite/CRA React.
  • Crear Mocks basados en los contratos de Spring.
📋 PO/QA/Encuestas
  • Análisis del flujo del alumno (QR -> Encuesta).
  • Preparar tableros Trello/Jira.
  • Definir la matriz de roles y permisos.

Semanas 3-6: Construcción de módulos autónomos

🗄️ BBDD (Mongo)
  • Despliegue local/Atlas de la base.
  • Poblado masivo de datos falsos de colegios.
  • Primeras "Aggregation pipelines" de prueba.
⚙️ Backend (Spring)
  • CRUDs de Centros, Cursos y Usuarios.
  • Conexión con Mongo Data.
  • Autenticación base y generación de JWT.
🖥️ Frontend (React)
  • Maquetar pantallas (Login, Admin Dashboard).
  • Desarrollar componentes visuales.
  • Usar Axios con datos Mocks (falsos).
📋 PO/QA/Encuestas
  • Desarrollar lógica de generación QR en Front.
  • Maquetar la vista pública de encuesta.
  • Pruebas de API con Postman.

Semanas 7-9: Conexión, estadísticas y pulido

🗄️ BBDD (Mongo)
  • Optimizar índices de búsqueda.
  • Cálculo de medias desde la propia base de datos.
⚙️ Backend (Spring)
  • Proteger endpoints según Rol y Centro.
  • Crear generador de CSV y estadísticas consolidadas.
🖥️ Frontend (React)
  • Eliminar Mocks. Enganchar React con Spring real.
  • Gestión de estados (Loaders, Toasts de éxito).
📋 PO/QA/Encuestas
  • Integrar librerías de gráficos (Chart.js/Recharts).
  • Pruebas End-to-End (E2E). Intentar romper el sistema de roles.

Semanas 10-12: Empaquetado y Cierre

🗄️ BBDD (Mongo)
  • Limpieza de documentos "basura" de las pruebas.
  • Diccionario de datos NoSQL finalizado.
⚙️ Backend (Spring)
  • Limpieza de código y logs.
  • Documentación de API entregada y generacion del JAR.
🖥️ Frontend (React)
  • Build optimizado de React preparado para servir.
  • Solución de warnings en consola y responsive checks.
📋 PO/QA/Encuestas
  • Manuales de usuario creados.
  • Presentación final ensayada.

Filosofía de Trabajo: Oleadas

Vosotros sois la Oleada 1. El objetivo no es hacer 1.000 funcionalidades a medias, sino dejar unos cimientos técnicos excelentes con React y Spring para la siguiente promoción.

🏛️

Diseño y API son Ley

Cualquier cambio estructural en Mongo o en las rutas de Spring debe ser debatido y aprobado por todos.

💎

Calidad > Cantidad

Un módulo de encuestas y CSV perfecto vale más que añadir notificaciones rotas o chats a medias.

🗣️

Comunicación Diaria

Si Spring Boot cambia una respuesta JSON, React dejará de funcionar. Hablad a diario.
